# New LPBF - Model

## 最终方案（汇总版）
### 数据介绍
* 通过基于 OpenFOAM 的求解器生成；
* 长方体场景，规则立方体网格（三个维度的范围、网格大小也不完全一致）；
* 同一样本时间间隔保持不变，不同样本之间也会有所差别；
* 三个维度网格数量约为 112 * 80 * 40（实际数据不完全一致，但比例差不多）；
* 不同数据通过修改激光功率、激光半径、激光速度生成；
* 物理坐标范围也会有所不同（比如 0-0.56 / 0.2-0.76 等等）；

### 所选 Baseline 介绍
* PhysGTO：GNN + Transformer 架构的算子学习模型（参考`references/An Efficient Graph-Transformer Operator for Learning Physical Dynamics with Manifolds Embedding.pdf`）；
* GNN 用于汇总局部信息，Transformer 用于获取全局信息，达到信息汇总的效果；
* Transformer 架构中使用的可学习 query，将复杂度降至线性；
* 是一个比较前沿的通用模型（适用于很多物理仿真场景，并不在某个领域专精）；
* 当前预测的物理场主要是温度场、气体体积分数场 alpha.air（决定金属和空气的分界）；后续考虑加入液态金属体积分数场（gamma.liquid，决定熔池边界）；暂时不考虑速度场
* 原版 physgto 模型代码见 `src/model/physgto.py`；改进版本见 `src/model/physgto_attnres_multi_v3.py`（没有摆脱 physgto，也是通用模型）

### 初步改进方案（LPBF 过程专用算子学习模型）
* 一定是算子学习模型，能够适用于不同规模、不同空间分辨率的规则立方体网格（适用于我们的数据），需要考虑是否继续使用 GNN（可能在规则网格上表现更好）；
* Transformer 可以考虑保留，如果 mamba 模型或者其他前沿的模型能够有更好效果的话也可以考虑使用或者整合，我现在已经用上的注意力残差方法也可以继续使用；但是使用的话，我更加倾向于每个物理场有单独的分支，然后在合适的地方进行 cross-attention 来获取信息。
* 因为是针对 LPBF 过程的专用模型，所以我们一定要参考 LPBF 过程的物理原理以及求解器采用的一些方法。比如要格外重视激光这个唯一驱动，现在的代码只是将激光有关的参数作为普通的 condition 变量传入（对于激光作用的重视程度远远不够），我觉得需要让模型对激光位置、能量强度有一个比较充分的认识，在进行前向预测的过程中或许会更加准确；
* 当前 dataset 文件中对于坐标、时间都采用了归一化操作，我觉得这种操作虽然有利于数值稳定，但是并不利于模型感知实际的物理时间、物理位置，也就不利于去预测某个位置会有多少的激光强度，也就不利于后续的预测；
* 在对激光的参数有了一定的感知之后，我们还需要向模型中嵌入更多与控制方程相关的物理信息，LPBF 过程的 NS 方程右侧有很多源项；我们要做的不是像做仿真一样严谨正确的求出这些源项，而是根据源项的性质、涉及的物理场，设计合理的、对应的网格架构，能够可解释的、较为正确的反演出这些源项，将其组合起来预测出正确的物理场；对于求解器其他的一些技巧（控制方程以外的）也完全可以拿过来使用（比如激光追踪算法）；
* 我的多物理场包含界面场（VoF方法表示）以及温度场；对于界面场，我们更加关注值在 0.5 附近的区域（比起物理量预测准确，更希望能够将分界线预测准确）；对于温度场肯定就是物理值要准确一些了，这里我觉得要注意进行区分；但是我每次预测物理场也可能会变化，可能这次预测温度场和气体体积分数场，下次就是气体体积分数场和液态金属体积分数场；
* 当然上述只是我的一些建议，如果你有更好的想法，能够创新出一个全新的架构（具有颠覆性意义的，类似于 Transformer），也可以进行尝试；

### 最终目标
* 我的求解器（参考`references/Multi_material_LPBF_Method.pdf`）主要采用的就是 CFD-DEM 这类前沿的方法（都差不多），VoF 方法是OpenfOAM 自带的；多材料框架可以参考附件中 Tang 的论文（参考 `references/Tang-multi-material.pdf`）。（我的求解器比较先进，支持多材料、振荡匙孔，但是我目前采用的数据只有单材料，未来会拓展到多材料，但是两种情况在控制方程上并没有太大差别，暂时不用考虑这一点）；
* 你需要先弄清楚 LPBF 物理过程以及求解器的细节；
* 广泛搜索调研机器学习领域的前沿文献内容，有合适的前沿的模型 / 机制（比如 LNN、注意力残差连接）可以引入；
* 广泛搜索调研 AI4Sci 领域的前沿文献内容，有合适的方法也可以进行引入（毕竟我们是在做物理场的重构，其严谨性、科学性还是应该保持的），但是我们不考虑传统 PINN 中在损失函数中添加 PDE 损失的做法；
* 我们真正需要的是一个将 LPBF 原理融合到模型架构中，且应用了机器学习领域中前沿的模型/算法的专用物理过程代理模型（physgto 现有的 GNN + transformer 架构不是必须保留）
* 先不用修改代码，先帮我设计一版合适的方案（主要是模型架构，当前的训练策略如果需要调整也可以一并指出）；
* 你尽管设计，需要增加传入的参数或者改变传入方式都可以满足！
* 注意，我上面的描述中举的例子仅供参考，不是一定要用或者只能使用！
